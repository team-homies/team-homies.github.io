# HOHOBOOK (도서 관리 프로그램)
## 1. 주제 선정
- 요구사항
    - 새 책 입력하여 저장하는 기능
    - 저장된 책 리스트 출력 기능
    - 대여 기록 현황 확인 기능(현재 도서가 대여중인지  / 누가 대여했는지 )
- 조건
    - 클라이언트와 서버 분리
    - 껐다 켜도 저장 기록 확인 가능
    - go 언어 쓰기
- 일정
    - 1/30(화) - 1/31(수) : 설계, 작업 배분 끝내기 (2일)
    - 2/1(목) - 2/6(화) : 프로토타입 완성 (6일)
    - 2/7(수) - 2/10(토) : 디버깅 및 보완 (4일)
    - 2/11(일) - 2/12(월) - 테스트 기간 (2일)
- 규칙
    - 하루에 한 번씩 노션 스크럼하기
    - *주의* 작업은 학원 수업 시간 외에
      
## 2. 기능정의 및 기술정의

    - 룰정의
        - 깃허브 커밋메시지, PR규칙

    - 클래스다이어그램 설계

    - api 명세서

    - 틀 만들기

        - gorm
        - mysql
        - go

### 2) 클라이언트

    - cui 만들기
    - api 호출

### 3) 서버

    - 도서 리스트 조회
        - API 구현
    - 도서 상세정보 조회
        - API구현
    - 대여자 조회
        - API 구현
    - 도서 정보 수정
        - API 구현
    - 도서 등록/삭제
        - API 구현
    - 도서 대출/반납

    - 헬스체크 API 구현

### 4) DB

    - 테이블생성+ DB환경

### 5) 테스트 / 디버깅

    - 디버깅
    - 테스트 케이스 문서화

### 6) 기술스택
    - golang : 기능 구현 시 사용되는 언어

### 7) 환경세팅
